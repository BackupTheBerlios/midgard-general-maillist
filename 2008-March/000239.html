Hallo Malte,

Malte Thoma schrieb:
> Wie wäre es mit einer c++lib und einer wxpython Oberfläche? Damit wäre
> man völlig Betriebssystemunabhängig, oder?

Noch Betriebssystemunabhängiger als mit Gtkmm und C++? (Gtkmm läuft
unter Windows, Linux & Co. (e.g. Solaris), MacOS X und einer Menge
embedded derivaten)

 ;-)

Aber sollte sich jemand finden, der Spaß daran hat, die Magus GUI unter
python (egal ob mit wx, gtk oder Qt oder ...) neu zu programmieren, so
verspreche ich, die Anbindung der libmagus (das was unter der Haube von
Magus 0.9+ arbeitet) an python (oder Ruby, oder Visual Basic oder ...)
zu machen.

Ich glaube, dass die Oberfläche das schwächste Teil von Magus 1.1 ist.
Immerhin ist sie seit 0.8 nicht mehr überarbeitet worden. Aber ich werde
nicht noch einen Port auf ein anderes Widget Set machen. Wahrscheinlich
zum Glück für uns alle.

Was nicht heißt, dass ich nicht innerhalb der nächsten Wochen eine neue
Windows Version (1.1) und eine Visual-Studio Variante der libmagus
veröffentlichen werde. Vielleicht wechsele ich ja ganz auf VC8 (auch für
die Gtkmm-GUI) - immerhin ist der Compiler jetzt ja kostenlos verfügbar
und weit verbreitet. Das könnte die Anzahl der Leute, die Magus
kompilieren können verzehnfachen ;-) und damit auch die Zahl der Leute,
die daran mitentwickeln können.

Liebe Grüße
  Christof

PS: Sieht so aus, als wäre meine neue Anstellung schlecht für meinen
Idealismus gewesen - ich erschwere die Verwendung von nicht freien Tools
nicht mehr ;-)
